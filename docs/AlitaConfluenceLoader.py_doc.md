# AlitaConfluenceLoader.py

**Path:** `src/alita_sdk/langchain/document_loaders/AlitaConfluenceLoader.py`

## Data Flow

The data flow within the `AlitaConfluenceLoader` class primarily revolves around the processing of images and documents retrieved from Confluence. The data originates from Confluence links, which are fetched using HTTP requests. The content retrieved can be in various formats such as PDFs, images, or SVGs. Depending on the format, the content undergoes different processing steps. For instance, PDFs are converted to images, and SVGs are rendered to PNG format. The processed images are then passed to a language model (LLM) for analysis, which generates textual descriptions or specifications based on the image content. The final output is a detailed text generated by the LLM, which is returned to the caller.

Example:
```python
response = self.confluence.request(path=link, absolute=True)
if response.status_code == 200 and response.content:
    images = convert_from_bytes(response.content)
    for image in images:
        result = self.__perform_llm_prediction_for_image(image)
        text += f"Page {i + 1}:\n{result}\n\n"
```
In this snippet, the PDF content is fetched, converted to images, and each image is processed by the LLM to generate text.

## Functions Descriptions

### `__init__`
The constructor initializes the `AlitaConfluenceLoader` with optional parameters such as `bins_with_llm`, `prompt`, and `llm`. It sets up the instance variables and removes these keys from the kwargs before calling the superclass constructor.

### `__perform_llm_prediction_for_image`
This private method converts an image to a byte array, encodes it in base64, and sends it to the LLM for prediction. The LLM returns a textual description of the image content.

### `load`
This method overrides the superclass's `load` method to map the `content_format` parameter to a specific `ContentFormat` enum value before calling the superclass method.

### `process_pdf`
This method processes a PDF link by converting its content to images and using the LLM to generate text for each image. If `bins_with_llm` is False or `llm` is not provided, it falls back to the superclass's method.

### `process_image`
This method processes an image link by fetching the image content and using the LLM to generate a textual description. It falls back to the superclass's method if `bins_with_llm` is False or `llm` is not provided.

### `process_svg`
This method processes an SVG link by rendering it to a PNG image and using the LLM to generate a textual description. It falls back to the superclass's method if `bins_with_llm` is False or `llm` is not provided.

## Functional Flow

1. **Initialization**: The `AlitaConfluenceLoader` is initialized with optional parameters for LLM integration and custom prompts.
2. **Loading Content**: The `load` method is called with parameters specifying the content format. It maps the format to a `ContentFormat` enum and calls the superclass's `load` method.
3. **Processing PDFs**: The `process_pdf` method fetches PDF content, converts it to images, and uses the LLM to generate text for each image.
4. **Processing Images**: The `process_image` method fetches image content and uses the LLM to generate a textual description.
5. **Processing SVGs**: The `process_svg` method fetches SVG content, renders it to a PNG image, and uses the LLM to generate a textual description.
6. **LLM Prediction**: The `__perform_llm_prediction_for_image` method handles the conversion of images to byte arrays, encoding to base64, and sending to the LLM for prediction.

## Endpoints Used/Created

The `AlitaConfluenceLoader` interacts with Confluence endpoints to fetch content. The specific endpoints are not hardcoded but are constructed dynamically based on the provided links. The HTTP requests are made using the `confluence.request` method, which handles the communication with Confluence's API. The responses are processed to extract content, which is then converted and analyzed as described in the functional flow.
