## Data Flow

The data flow within `AlitaCallback.py` revolves around handling various callback events for an Alita agent. The data originates from different events such as chain start, tool start, LLM start, etc., and is processed through corresponding callback methods. These methods log the events, update the callback state, and manage the status messages displayed in Streamlit. The data is primarily in the form of dictionaries and lists, which are manipulated and transformed to update the callback state and log information.

For example, in the `on_tool_start` method, the data flow can be illustrated as follows:

```python
    def on_tool_start(self, *args, run_id: UUID, **kwargs):
        if self.debug:
            log.info("on_tool_start(%s, %s)", args, kwargs)

        tool_name = args[0].get("name")
        tool_run_id = str(run_id)
        payload = {
            "tool_name": tool_name,
            "tool_run_id": tool_run_id,
            "tool_meta": args[0],
            "tool_inputs": kwargs.get('inputs')
        }
        payload = json.loads(json.dumps(payload, ensure_ascii=False, default=lambda o: str(o)))
        self.callback_state[tool_run_id] = self.st.status(f"Running {tool_name}...", expanded=True)
        self.callback_state[tool_run_id].write(f"Tool inputs: {kwargs.get('inputs')}")
```

In this example, the data flows from the event arguments (`args` and `kwargs`) to the `payload` dictionary, which is then used to update the `callback_state` and display the status in Streamlit.

## Functions Descriptions

### `__init__(self, st: Any, debug: bool = False)`

The constructor initializes the callback handler with the Streamlit instance and a debug flag. It sets up the initial state, including the callback state, token counters, and the current model name.

### `on_chain_start(self, *args, **kwargs)`

This method is called at the start of a chain. If debugging is enabled, it logs the event.

### `on_chain_end(self, *args, **kwargs)`

This method is called at the end of a chain. If debugging is enabled, it logs the event.

### `on_chain_error(self, *args, **kwargs)`

This method is called when there is an error in the chain. If debugging is enabled, it logs the event.

### `on_tool_start(self, *args, run_id: UUID, **kwargs)`

This method is called at the start of a tool. It logs the event if debugging is enabled, creates a payload with tool information, and updates the callback state to display the status in Streamlit.

### `on_tool_end(self, *args, run_id: UUID, **kwargs)`

This method is called at the end of a tool. It logs the event if debugging is enabled, updates the callback state with the tool output, and marks the tool as completed in Streamlit.

### `on_tool_error(self, *args, run_id: UUID, **kwargs)`

This method is called when there is an error in a tool. It logs the event if debugging is enabled, updates the callback state with the error message, and marks the tool as errored in Streamlit.

### `on_agent_action(self, *args, **kwargs)`

This method is called when an agent takes an action. If debugging is enabled, it logs the event.

### `on_agent_finish(self, *args, **kwargs)`

This method is called when an agent finishes. If debugging is enabled, it logs the event.

### `_handle_llm_start(self, serialized: Dict[str, Any], messages: List[List[BaseMessage]] | List[List[str]], *, run_id: UUID, parent_run_id: Optional[UUID] = None, tags: Optional[List[str]] = None, metadata: Optional[Dict[str, Any]] = None, **kwargs: Any)`

This method is called at the start of an LLM call. It delegates to `_handle_llm_start` to handle the event.

### `on_chat_model_start(self, *args, **kwargs)`

This method is called at the start of a chat model call. It delegates to `_handle_llm_start` to handle the event.

### `on_llm_new_token(self, *args, run_id: UUID, **kwargs)`

This method is called when a new token is generated by the LLM. If debugging is enabled, it logs the event and updates the callback state with the new token.

### `on_llm_error(self, *args, run_id: UUID, **kwargs)`

This method is called when there is an error in the LLM. If debugging is enabled, it logs the event, updates the callback state with the error message, and marks the LLM call as errored in Streamlit.

### `on_text(self, *args, **kwargs)`

This method is called when text is received. If debugging is enabled, it logs the event.

### `on_llm_end(self, response: LLMResult, run_id: UUID, **kwargs)`

This method is called at the end of an LLM call. If debugging is enabled, it logs the event and updates the callback state to mark the LLM call as completed in Streamlit.
