Fetches Azure DevOps work item hierarchy and enriches with Acceptance Criteria (AC) data. Returns the filename in a list format.

**Input:**
```json
{"id": 9, "expand": true, "output_name": "my_run"}
```

**Fields:**
- `id` (integer, required): Work item ID. Example: `9`, `42`, `123`
- `expand` (boolean, required): Hierarchy expansion. `true` = include descendants, `false` = epic only
- `output_name` (string, optional): Custom filename identifier. Auto-generated if not provided

**Output:** `["9_acs_my_run.json"]`

**Filename:** Format: `{id}_acs_{output_name}.json` or `{id}_acs.json`

**File Created:** A JSON file with hierarchy enriched with extracted ACs, AC coverage mappings and relationships. The `expand` flag controls child inclusion.
