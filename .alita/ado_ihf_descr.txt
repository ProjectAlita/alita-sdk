Fetches Azure DevOps work item hierarchy and saves to JSON file. Returns the filename in a list format.

**Input:**
```json
{"id": 9, "expand": true, "output_name": "my_run"}
```

**Fields:**
- `id` (integer, required): Work item ID. Example: `9`, `42`, `123`
- `expand` (boolean, required): Hierarchy content. `true` = include all descendants, `false` = only item
- `output_name` (string, optional): Custom filename identifier. Auto-generated if not provided

**Output:** `["9_feature_my_run.json"]`

**Filename:** Format: `{id}_feature_{output_name}.json` or `{id}_feature.json`

**Files Created:** JSON with hierarchy data, and TXT with visual tree format. The `expand` flag determines whether child work items are included in the saved JSON file.
